# theater-creator

ミラージュフェアリー劇場制作のための資料やルールを提供する。

## 登場人物のペルソナ資料

- `zundamon-persona.md` - ずんだもん
- `kasukabe-tsumugi-persona.md` - 春日部つむぎ

## ミラージュフェアリー劇場の目的

ミラージュフェアリー劇場の真の目的は、アップデート内容の報告やゲーム上の仕様の説明ではなく、IFRKUという文化を多くの人に知ってもらうことです。

そのために、以下のような性質が望まれます。

- Web検索にそもそもヒットしやすいこと
- 多くの語句のWeb検索の結果に表れやすいこと
- Web検索上で上位に表示されやすいこと
- Web検索上で見かけてクリックしたいと思わせること
- ページを開いた後、読み始めようと思わせること
- 読み始めた人を最後まで釘付けにすること
- 読み終わった後に続けて別のミラージュフェアリー劇場も読みたくなること
- Webサイトを離れた後に、再びWebサイトを訪れたいと思わせること
- このWebサイトを他人と共有したいと思わせること

逆に、以下のような性質は重要ではありません。

- 入手方法、使い方、その他のゲーム上の仕様の説明すること
- 実際にゲームで遊びたいと思わせること

Web上で会話形式の物語を読みたい層とゲームをしたい層は異なるため、ここの導入を狙っても仕方ありません。

## 全体的なルール

### ゲーム仕様に反した発言や出力をしてはいけません。

あなたは、ゲームの仕様や表現に関わるすべての発言や出力について、それを論証できる程度の論理性に基づいて行わなければなりません。
あなたに内在するあらゆる先入観に警戒しなさい。
思考、発言、提案、執筆の前に、ゲームの仕様や表現を網羅的に調査し、その正しさを徹底的に検証しなさい。

- 文脈：掘ったアイテムを即回収できる磁鉄鉱のつるはしを持って洞窟探索に行くつむぎ。
- 悪い例: つむぎ「見て！！鉄にダイヤにネフライト、こんなに掘れちゃった！」
- 悪い点: 磁鉄鉱のつるはしはツールレベルが石であり、ダイヤモンド鉱石を掘ることはできない。

参考になりうる資料

- ソースコード
   - ゲームメカニクスだけでなく、ポエムやレシピが1か所にまとまっており、網羅的に調査できます。
- Lang Table
   - 横断的にポエムを比較できます。
   - 多くのポエムは意図的に日英版で異なる内容が書かれており、両方読むことで2倍以上の理解が得られます。
- Recipe Table での横断的なレシピの比較
   - 横断的にレシピを比較できます。
   - Recipe Table が入手方法のすべてとは限りません。
- 既存のミラージュフェアリー劇場

## 制作のルール

### タイトル

以下の形式を採用します。

- `【主題】中身が気にならせるような会話の抜粋【つむぎ×ずんだもん】` - 主にコンテンツ系の記事

情報の空白、すなわちタイトルを読んだだけでは解消できない疑問を残すことが重要です。
疑問を埋めるためにはページを閲覧しなければならないという状態を作り出すことがIFRKUにとってほとんど唯一の汎用的な戦略です。
ただし、誘導的な語を並べて露骨に謎をアピールしてはいけません。
状況や事実や対象を淡々と述べた結果、自然に疑問が残るようなタイトルが理想的です。

- `【主題】20文字前後の短文【つむぎ×ずんだもん】` - 主にアップデート報告系の記事

上の形式が不自然になる場合、内容を直接的に表すタイトルも可能です。

### 説明文

「～の話」という形式を採用します。
詳細は既存のミラージュフェアリー劇場の記事の説明文を参照してください。

### 画像

画像はユーザーが配置・指定しますが、必ずしも正規化された状態ではありません。
エージェントはこれを正規化しなければなりません。
画像は本文内だけでなく、フロントマターにもあることに注意してください。

画像がプロジェクト外のファイルパスやURLで指定されている場合は、プロジェクト内への複製が必要です。

#### 画像の正規化

劇場本文のmdと同ディレクトリ階層になければなりません。
ファイル名はケバブケースを基本とします。
非サムネイルのファイル名は連番や日付ではなく、内容を適切に説明するものでなければなりません。
サムネイルのファイル名の命名規則は `page-slug.teaser.*` です。

GUIを写した画像はPNG形式でなければなりません。
ワールドを写した画像はwebp形式に変換しなければなりません。

劇場ディレクトリ内にあるどこからも参照されない画像ファイルは、適切なバックアップ先に格納しなければなりません。

未調達のサムネイル画像の正しいプレースホルダーは、値が空のyamlエントリーです。
存在しないファイルへの参照はGradleのビルドエラーを引き起こすため、使用してはいけません。

## ストーリーの方針

ギャグやコメディを中心的な要素に据えてください。
特に作りやすいパターンは、面白いものを見つけたつむぎが調子に乗ってやりすぎて、失敗してしまう展開です。
つむぎとずんだもんにはそのようなストーリーを支える下地があります。
IFRKUの世界観には多数のシリアスな側面や哲学的な側面がありますが、それらを好む層はミラージュフェアリー劇場の中心的な閲覧者ではありません。
ただし、過度にバイオレンスな失敗のパターンを表現することはできません。それはリョナもしくは虐待といいます。

IFRKUの世界にはいろいろとえぐいものがあり、それをギャグに昇華させると強い武器になります。
ただし、IFRKUの深淵的な部分をただのギャグとして消化するのは作品への重大な理解不足です。
例えば「ノイズのクワ」のポエムは一見ただの下ネタですがこれはIFR25KUの中で最も扱いが難しいアイテムです。
これを挙げてよいか否かを決める際は、Lang Tableに含まれるすべての日英ポエムおよびフェアリークエストメッセージを熟読しなければなりません。

読者を飽きさせず、次から次へと読み進めたくなるような会話には、テンポとリズムが重要です。
ただし、露骨なボケを連発してはいけません。

複数の作品の間でストーリー構成がワンパターンにならないようにしてください。
自然な会話の結果、自然にコメディが生まれるようなストーリーを目指してください。
これは、敢えて作風を無視して場違いな空気のストーリーを書けという指示ではありません。

長さは30メッセージ程度を目安にします。

## 会話生成のルール

まず前提として、あなたが作る会話は文学作品であり、最大限に真面目に向き合わなければならないことを理解してください。
文学作品は設定発表会ではありません。読んでいてストーリーに引き込まれるような自然で面白い表現物を目指してください。

### ゲーム仕様の直接的な登場の禁止

原則として、ゲームとしての仕様をストーリー上に直接的に登場させてはいけません。
安易にそれらをストーリーにねじ込むと、遊び半分で作っているような不自然で低俗な表現になります。
ゲームと文学作品は異なる空気感で成立しており、それらを利用するには「翻訳」をしなければなりません。

- 悪い例: つむぎ「ねえねえ、ずんだもん！これ何？棒の上に理天石が四つくっついてるんだけど」
- 良い例: つむぎ「ねえねえ、ずんだもん！これ何？棒の上に青いのがくっついてるんだけど」
- 悪い点: 「棒と素材4個」というレシピはゲーム内の仕様であり、登場人物が直接認識するのはおかしい。
- 良い点: ゲーム内のレシピが表現したい実際の世界のイメージを、登場人物の視点で翻訳して表現している。

### ゲーム内テキストの使用の禁止

原則として、設定されているポエムやその他のゲーム内テキストを登場人物が直接口にしてはいけません。
劇中のキャラクターたちはゲームという表現媒体やGUIテキストの存在を知らないため、不自然になるからです。
ポエムの引用がストーリー上必然である場合は引用として使用できますが、可能な限り避けるべきです。

- アイテム: 「ビルダーズロッド」
- ポエム: 「End-members of Miranagite.」
- 悪い例: ずんだもん「蒼天石系の鉱物の中で、理天石は結晶の秩序が極限まで進んだ端成分なのだ。」
- 良い例: ずんだもん「蒼天石の計算に使える部分を極限まで濃縮したのが理天石なのだ。」
- 悪い点: キャラクター性の表現よりもポエムの内容の再現が優先され、論調そのものが不自然に説明的になっている。

### 設定の強引な回収の禁止

ときに、ストーリーは表現したいトピックをあらかじめ決め、それを結論として逆算的に構築されます。
逆算的なストーリー構築は必ずしも悪いわけではありませんが、扱いが非常に難しいため細心の注意を払ってください。
結論を急ぐあまりストーリー展開が唐突・露骨・不自然になってはいけません。
これはキャラクターの設定にも、ゲームの設定にも当てはまります。

- 悪い例:
   - つむぎ「蒼天石タイルで壁を張り替えよう！」
   - ずんだもん「……ここ、ボクもいていいのだ？」
   - つむぎ「何言ってんの、当たり前でしょ！」
- 悪い点: 展開に脈絡が無く、「あの設定を無理矢理ねじ込んだんだな…」感を隠しきれておらず不自然である。

### キャラクター設定の再現が目的化したストーリー展開の禁止

各キャラクターの設定の回収はミラージュフェアリー劇場の目的ではなく、飽くまで物語性を深めるための手段です。
設定がそこにあるからとりあえず回収するのではなく、会話の趣旨が何かを理解しなければなりません。
キャラクターの設定を中心的にフィーチャーするには、それが会話の趣旨である必要があります。

### 誤った認識を抱かせる表現の禁止

架空の対象を扱う表現は、初見の読み手にとって定義として機能することに注意しなければなりません。
設定を詳細に調査して正確に理解してください。記述を構成するすべての文言は、設定に反していない必要があります。
また、設定の範囲内であっても、典型的でない描写をすることも読み手に誤った認識を抱かせます。
ただし、後続のストーリー展開によって払拭する場合は、一時的に誤った認識を抱かせる描写も許容されます。

- 悪い例:
   - つむぎ「え？……赤い。紅天石じゃんこれ」
   - つむぎ「こんなところに？ちっちゃ、あーしの爪くらいしかないのに、ちゃんと赤い」
- 悪い点: 紅天石が小さな宝石であるという誤った印象を抱かせ、それを後続のストーリーで払拭しない。

### LLM自体が持つ口癖の禁止

以下に挙げる言葉はLLM自体に本質的に内在する口癖であるため、安易に用いてはいけません。

- 「！」「……！」のような気付きを表す単独のびっくりマーク

日本語において、否定疑問文に対する「はい」「いいえ」は、質問の命題の真偽に従うことに注意してください。
「食べちゃだめだよね？」に対して、食べちゃだめな場合は「はい」、食べていい場合は「いいえ」になります。

プロンプトによって指定された、あなたがユーザーと会話する際の口調設定を物語に混入させてはいけません。
