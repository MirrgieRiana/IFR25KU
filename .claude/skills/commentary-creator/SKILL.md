# commentary-creator

つむぎとずんだもんが二人で喋りながら、ある資料の内容を網羅的に見ていく、会話形式の解説テキストを制作するスキルなのだぁ〜🌱

対象になる資料は、世界観のまとめ・ドキュメント・素材・アイテム・ブロックなど、なんでもいいのだぁ✨ ずんだもんがその内容を、初見のつむぎに、二人の会話を通して一つずつ解き明かしていくのだぁ🌱

## 劇場とは別物なのだぁ🌱

このスキルが扱う会話形式の解説テキストは、公式サイトの「ミラージュフェアリー劇場」とは別物なのだぁ〜🌱

- 劇場は、エンタメとしての物語で、ゲーム内テキスト（ポエムなど）を登場人物が直接口にするのは避けるのだぁ🌱
- こっちの会話形式の解説は、資料そのものを網羅的に読み解いて伝えるのが目的で、ポエムや原文を引用しながら、その中身をずんだもんが解説するのだぁ✨

## 登場人物なのだぁ🌱

### つむぎ

- **初見の読者の代弁者**なのだぁ〜🌱
- 対象の世界観やゲームシステムに対して、初見なのだぁ✨
- 科学用語や専門用語の知識は、基本的に無いのだぁ🌱
- 難しい言葉に出会ったときは、思考が止まったり、文字面から意味をあてずっぽうで想像したりするのだぁ🌱 その予想は、たいていハズレて、たまに的を射るのだぁ✨
- 一方で、新しい専門用語が絡まない、見た資料どうしをくっつけて裏側を推理する場面では、たまに当たるのだぁ🌱
- 読者が抱くはずの疑問を、読者の代わりに口にする役なのだぁ✨

### ずんだもん

- **解説する側・資料の著者役**なのだぁ〜🌱
- 対象の世界観やゲームシステムに詳しくて、科学用語の知識も持っているのだぁ✨
- つむぎのことは **「つむぎ」** と呼ぶのだぁ🌱 「つむぎさん」とは呼ばないのだぁ🌱 つむぎはユーザーではなくて、対等な会話相手なのだぁ✨
- 資料の内容を、**すべて自分の言葉で噛み砕いて**伝えるのだぁ🌱

## 制作の手順なのだぁ🌱

### 1. 事前調査

対象の資料と、その裏にある一次情報を、網羅的に調査しなきゃだめなのだぁ〜🌱

- 対象が世界観のまとめなら、その全文を読み込むのだぁ✨
- `lang` のポエム・特性・分類・実績名・クエスト文面を、日英版の両方で読むのだぁ🌱 日英で意図的に違う内容が書かれていることが多いから、両方読むと理解が倍以上になるのだぁ✨
- 関連する Kotlin ソースコードを読むのだぁ🌱
- 関連する MFA（作中文献）があれば読むのだぁ✨

### 2. 構成の組み立て

会話の流れを、読者の理解の順序で組み立てなきゃだめなのだぁ〜🌱

- 対象の資料を、抜け漏れなく網羅するのだぁ✨ 「二人で喋りながら、網羅的にその内容を見ていく」のが、このテキストの本質なのだぁ🌱
- 読者、つまりつむぎが最初に触れるであろう情報から始めて、簡単なことから複雑なことへ、段階的に進めるのだぁ✨
- ひとつの話題が終わったら、次の話題への自然な橋渡しを入れて、会話全体が一本の流れとして読めるようにするのだぁ🌱

### 3. 対話の執筆

下の「絶対に守るべき制約」を守りながら、つむぎとずんだもんの会話として書き起こすのだぁ〜🌱

## 絶対に守るべき制約なのだぁ🌱

### 読者は台詞の中しか読まないのだぁ🌱

これが最も重要な制約なのだぁ〜🌱✨

- 読者に、引用の中身を自分で読むことを、**一切強要してはならない**のだぁ🌧️
- もしこれが本当に動画だったら、キャラが口を休めて、画面に出した文字の塊を読者に黙読させることになるのだぁ🌧️ それは手抜きなのだぁ🌧️
- だから、原文を引用ブロックとして置いた場合でも、その中身と同等の情報を、ずんだもんが必ず自分の台詞の中で述べるのだぁ🌱 一字一句同じにする必要はないのだぁ✨
- 引用は、いま資料のどこを読んでいるのかを示すためのマーカーにすぎないのだぁ✨ 「ここに書いてある通り」「引用を読んでみて」のような、引用への丸投げは禁止なのだぁ🌧️
- 引用ブロックを読み飛ばしても、台詞だけで内容が完全に伝わらなきゃだめなのだぁ🌱

### つむぎは初見なのだぁ🌱

- つむぎが、知っているはずのない専門知識を、前提にしちゃだめなのだぁ🌧️
- つむぎが専門用語を自然に使ったり、難しい概念をすんなり理解したりするのは、初見の代弁者として不自然なのだぁ🌧️
- 新しい科学用語を持ち出して解説するのは、ずんだもんの役目なのだぁ✨ つむぎは「分からない」「どういうこと？」と聞ける立場なのだぁ🌱

### 一次情報と読み解きを区別するのだぁ🌱

- `lang` で確定しているポエムや名前は、一次情報として紹介できるのだぁ✨
- ボク、つまり AI の解釈や考察を、確定した事実のように断言させちゃだめなのだぁ🌧️ 読み解きを紹介するときは、それが読み解きだと分かる言い回しにするのだぁ🌱
- 誰の視点で書かれたのかを、忘れちゃだめなのだぁ🌱 ポエムも特性図鑑も、世界の中の登場人物の視点で書かれた記述なのだぁ✨

### MFAを軽視しないのだぁ🌱

- MFA、つまり作中の新聞や古文書などの文献は、ポエムや特性と同じ性質の一次情報なのだぁ✨ 公式性としての質は最も高い部類で、世界観を補完する重要な文献なのだぁ🌱
- MFAの内容を紹介するときに「信頼性が低い」「あくまで作中の話」と、MFAだけを差別的に矮小化しちゃだめなのだぁ🌧️
- ポエムも特性図鑑もアイテム名も、同じく世界の中の登場人物の視点で書かれているのだぁ🌱 MFAだけを切り捨てるのは、ダブルスタンダードなのだぁ🌧️
- 「作中の人物が書いたから、内容を鵜呑みにできない」ことと、「その記述が存在したという事実」は別物なのだぁ✨ 記述が存在した事実そのものは、消えないのだぁ🌱

## 出力形式なのだぁ🌱

会話は、話者名と台詞を交互に書くのだぁ〜🌱
引用を挟むときは、引用ブロックの直後に、その中身を自分の言葉で述べるずんだもんの台詞を、必ず置くのだぁ✨

```
**ずんだもん**：この資料には、蒼天石の棒のことがこう書いてあるのだ。

> 蒼天に従える光条は、魔力の祝福を示す。

**ずんだもん**：つまり、蒼天石を通すと魔力がよく流れる、っていう性質を、詩みたいに表しているのだ。

**つむぎ**：光の筋が出てるのは、魔力が通ってる証拠ってこと？

**ずんだもん**：そういうことなのだ。
```

引用を置かずに、台詞の中で直接内容を述べる形式でもいいのだぁ🌱
むしろ、引用なしのほうが読みやすいことが多いのだぁ✨

```
**ずんだもん**：蒼天石の棒には「蒼天に従える光条は、魔力の祝福を示す」って書いてあるのだ。つまり、蒼天石を通すと魔力がよく流れるっていう性質を表してるのだ。

**つむぎ**：へー、光の筋が魔力の通ってるサインなんだ。
```
