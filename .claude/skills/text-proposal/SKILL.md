# text-proposal

ゲームコンテンツの名前やポエムなどのテキストの候補を提案するための手順を提供するのだぁ〜🌱

このスキルは、候補を確定させるためのものじゃないのだぁ〜🌱
多様な候補を考案して、根拠とともに、おすすめ順でユーザーに提示しなきゃだめなのだぁ〜🌱

## 事前調査

提案の前に、対象の本質と、それが置かれる文脈を調査しなきゃだめなのだぁ〜🌱

対象そのものの性質や、位置付けを理解しなきゃだめなのだぁ〜🌱
物質、生物、地形などを問わず、その由来、構造、組成、原理、設計思想、物理・化学・地質・文化的な成り立ち、生態、世界観上の役割など、様々な面を調査して、そして考察しなきゃだめなのだぁ〜🌱

背景となる文脈も調査しなければならないのだぁ〜🌱
提案の発端となった Issue / PR や、関連する過去の議論を読まなきゃだめなのだぁ〜🌱

既存の類似物の記述も調査しなければならないのだぁ〜🌱
ソースコードには各種テキストデータが1か所にまとまっていて、Lang Table ではこれを横断的に比較できるのだぁ〜✨
同様の形状をもつものや、同種の素材で出来ているものを、クロスオーバーさせて調査しなきゃだめなのだぁ〜🌱
日英版は、かならず両方読んでほしいのだぁ〜🌱

## 制約

### 命名の制約

名前の候補は、次の制約を満たさなきゃだめなのだぁ〜🌱

基本的に、既存のMinecraftおよびMOD内の同種の名前と比べて、突飛じゃないことなのだぁ〜🌱
敢えて突飛な名前を採用してもかまわないけど、安易に付けると、何も考えずに適当に付けたように感じられちゃうのだぁ〜💧
既存の命名パターンの上に、意味のある発想を積み重ねてほしいのだぁ〜🌱
名前は、日本語版と英語版のペアで提示しなきゃだめなのだぁ〜🌱

### アイテムおよびブロックのポエムの制約

日本語版は20文字以内、英語版は40文字以内なのだぁ〜🌱
これは厳守の制約なのだぁ〜🌱
文字数は、一字ずつ数えて確認してほしいのだぁ〜🌱

日英を直訳で揃えるとは限らないのだぁ〜🌱
既存のポエムの書き分けを理解した上で、形式を選ばなきゃだめなのだぁ〜🌱

## 候補の考案

候補は、似たり寄ったりの構造や観点のテキストを大量に作るんじゃなくて、様々な点でなるべく多様にばらけるように、数多く発散させなきゃだめなのだぁ〜🌱✨
概要を記述してサブエージェントにも投げて、多様な形式の案を集めなきゃだめなのだぁ〜🌱

パターン数は、指定が無ければ、既定で、考案対象ごと、日英版ごとに、それぞれ 20 パターンなのだぁ〜🌱
生成した候補を、勝手に間引いたり、少数に選別して報告しちゃだめなのだぁ〜🌱

## 出力形式

考案対象ごと、日英版ごとに、箇条書きリストで出さなきゃだめなのだぁ〜🌱
英語版には、その直訳を右に括弧書きで付記しなきゃだめなのだぁ〜🌱
各案への説明は、既定では付けないのだぁ〜🌱
順序は、おすすめだと思う順に並べるのだぁ〜🌱

次は出力例なのだぁ〜🌱

```
- 最初の候補
- 次の候補
```

```
- First candidate（最初の候補）
- Next candidate（次の候補）
```

---

各案に説明の併記が求められた場合は、140字以上で、その案の良さや、相応しいと思う点を熱弁してほしいのだぁ〜🌱🔥
このときは、フォーマットは箇条書きリストじゃなくて、段落による候補と、箇条書きによる説明を並べなきゃだめなのだぁ〜🌱

```
最初の候補

- 最初の候補に関する説明です。

次の候補

- 次の候補に関する説明です。
```
