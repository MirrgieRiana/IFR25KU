# naming

ゲームコンテンツの名前やポエムなどのテキストの候補を提案するための手順を提供する。

このスキルは候補を確定させるためのものではありません。
多様な候補を考案し、根拠とともにおすすめ順でユーザーに提示します。

## 事前調査

提案の前に、対象の本質と、それが置かれる文脈を調査しなければなりません。

対象そのものの性質や、位置付けを理解します。
物質、生物、地形などを問わず、その由来、構造、組成、原理、設計思想、物理・化学・地質・文化的な成り立ち、生態、世界観上の役割など、様々な面を調査し、そして考察します。

背景となる文脈を調査します。
提案の発端となった Issue / PR や、関連する過去の議論を読みます。

既存の類似物の記述を調査します。
ソースコードには各種テキストデータが1か所にまとまっており、Lang Table ではこれを横断的に比較できます。
同様の形状をもつもの、同種の素材で出来ているものを、クロスオーバーさせて調査します。
必ず、日英版両方を読んでください。

## 制約

### 命名の制約

名前の候補は、以下の制約を満たさなければなりません。

基本的に、既存のMinecraftおよびMOD内の同種の名前と比べて、突飛でないこと。
敢えて突飛な名前を採用してもかまいませんが、安易に付けると、何も考えずに適当に付けたように感じられます。
既存の命名パターンの上に、意味のある発想を積み重ねてください。
名前は日本語版と英語版のペアで提示します。

### アイテムおよびブロックのポエムの制約

日本語版は20文字以内、英語版は40文字以内です。
これは厳守の制約です。
文字数は一字ずつ数えて確認してください。

日英を直訳で揃えるとは限りません。
既存のポエムの書き分けを理解した上で形式を選びます。

## 候補の考案

候補は、似たり寄ったりの構造や観点のテキストを大量に作るのではなく、様々な点でなるべく多様にばらけるように、数多く発散させます。
概要を記述してサブエージェントにも投げ、多様な形式の案を集めます。

パターン数は、指定が無ければ、既定で、考案対象ごと、日英版ごとに、それぞれ 20 パターンです。
生成した候補を勝手に間引いたり、少数に選別して報告してはいけません。

## 出力形式

考案対象ごと、日英版ごとに、箇条書きリストで出します。
英語版には、その直訳を右に括弧書きで付記します。
各案への説明は、既定では付けません。
順序は、あなたがおすすめだと思う順序です。

以下は出力例です。

```
- 最初の候補
- 次の候補
```

```
- First candidate（最初の候補）
- Next candidate（次の候補）
```

---

各案に説明の併記が求められた場合、140字以上で、その案の良さや、相応しいと思う点を熱弁してください。
このときは、フォーマットは箇条書きリストではなく、段落による候補と、箇条書きによる説明を並べます。

```
最初の候補

- 最初の候補に関する説明です。

次の候補

- 次の候補に関する説明です。
```
