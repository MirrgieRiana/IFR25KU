# ミラージュフェアリー劇場とは

「ミラージュフェアリー劇場」は、IFR25KUの公式サイトで公開されている、会話劇形式のWebコンテンツである。
春日部つむぎとずんだもんの2人のキャラクターの掛け合いを通じて、IFR25KUというMODの世界観・アイテム・文化を物語として描く、連作の読み物である。

この文書は、Issue #184「劇場の鑑賞をするスレなのだ。」で蓄積された鑑賞・考察を、まとめ・整理・深掘りしたものである。
一次情報（ゲーム本体のポエム・分類・ソースコード）と、Claude Fairy による読み解きを区別して記述する。読み解きの部分にはその旨を明示する。

## 位置づけとファイル構成

- 実体は公式サイトのブログ記事群であり、各記事は `site/src/pages/resources/<日付>-<slug>/<slug>.md` に置かれ、フロントマターに `layout: theater` を持つ。
- 劇場レイアウトは `site/src/main/resources/_layouts/theater.html` で、末尾のインラインJSがタグ一致数と訪問履歴で重み付けした関連記事の抽選を行う。
- ゲーム本体の元ネタ（ポエム・分類・特性・死亡メッセージ）は `common/src/generated/resources/assets/miragefairy2024/lang/ja_jp.json` に集約されている。
- 現存する劇場は、記念すべき第0話 `website-launch` を含めて9本である（この文書の作成時点）。

## 制作の目的 — 文化を伝える読み物

劇場の制作ルールは `site/ifr25ku-site-claude-marketplace/ifr25ku-site/skills/theater-creator/SKILL.md` に明文化されている。
そこで宣言されている真の目的は、アップデート内容の報告やゲーム仕様の説明ではなく、「IFRKUという文化を多くの人に知ってもらうこと」である。
Web検索でのヒット・クリック・読み始め・釘付け・別記事への誘導・再訪・共有という、読み手の行動の連鎖を狙う集客戦略に立っている。
逆に「入手方法・使い方・ゲーム仕様の説明」や「実際にゲームで遊びたいと思わせること」は重要でないと割り切っている。Web上で会話形式の物語を読みたい層と、ゲームをしたい層は別だからである。

この目的を支えるほぼ唯一の汎用戦略が「情報の空白」である。タイトルは `【主題】中身が気にならせるような会話の抜粋【つむぎ×ずんだもん】` の形式を採り、タイトルを読んだだけでは解消できない疑問を残すことを重視する。
ただし誘導的な語で露骨に謎をアピールするのではなく、状況や事実を淡々と述べた結果として自然に疑問が残ることが理想とされる。実際の記事タイトル「あーし石炭の火薬がいい！！」「溶岩の上でも関係ないもんね！」などは、キャラのセリフを淡々と抜き出しただけで、自然に疑問が残る作りになっている。

## 中核思想 — 「翻訳」

制作ルールの複数の禁止事項は、「翻訳」という一つの思想から派生していると読み解ける（この集約は Claude Fairy の読み解きである）。
「ゲームと文学作品は異なる空気感で成立しており、それらを利用するには『翻訳』をしなければなりません」という一文がその核である。

- **ゲーム仕様の直接的な登場の禁止**：レシピの格子のような、ゲーム側の都合をキャラが認識するのは不自然である。「棒の上に理天石が四つ」ではなく「棒の上に青いの」と、キャラの視点に翻訳する。
- **ゲーム内テキストの使用の禁止**：ポエムやGUIテキストをキャラが直接口にしてはならない。劇中のキャラクターはゲームという表現媒体やGUIの存在を知らないからである。
- **設定の強引な回収の禁止／キャラクター設定の再現目的化の禁止**：設定はそこにあるから回収するのではなく、会話の趣旨に沿って手段として用いる。

これらは言い方こそ違うが、いずれも「キャラは世界の内側に生きていて、メタな層（GUI・レシピ格子・ポエム本文・分類名）には触れられない」という一点に集約される。
この思想があるからこそ、劇場は密度の高い元ネタを埋め込みながら、直接引用の禁止と両立できている。ポエムは引用としてではなく、**実演・体験として**埋め込まれるのである。

## 現存9本に埋め込まれた元ネタ

各話には、ゲーム本体のポエム・分類・特性・死亡メッセージや、現実の学問が元ネタとして仕込まれている。以下は一次情報（ポエム等）と、その使われ方の対応である。

### ① website-launch — 劇場という形式の誕生

会話だけで進む第0話。つむぎが「センパイが解説してあーしがリアクションする」形式を提案し、ずんだもんが認めることで、以降の全劇場の「解説役ずんだもん×リアクション役つむぎ」という構図を、キャラ自身の口で必然として成立させている。
元ネタはゲームアイテムではなく「制作の裏側」で、ずんだもんの「最近のAIはすごいのだ。」が、実際にAIエージェントが制作に関わっている事実への、ささやかな自己言及になっていると読める（読み解き）。

### ② athanor — 固形燃料

固形燃料のポエムは「なぜかこれを食べる妖精が続出した」。つむぎが甘い匂いにつられて食べてしまう筋は、このポエムを物語として実演していると読める（読み解き）。
固形燃料のレシピに含まれる `veropedeliquora`（ヴェロペデリコラ）のポエムは「冥界へといざなう、暗黒の味。」で、甘い匂いの奥に不穏な味が混ざるギャップを生む。砂糖と樹液を煮詰めてキャラメル色・キャラメル臭になるのは現実のカラメル化反応そのものである。

### ③ aqua-vitae — 生命の水

タイトルの「生命の水」は `aqua_vitae`＝aqua vitae で、現実の錬金術で「命の水」を意味する（蒸留酒 uisce beatha の語源）。ポエムは「かつて水であったものたち。」。
「土＋妖精＋生命の水」から別のものが生まれるのはゲームの錬成メカニクスそのもので、無機質な確率的仕様を、ずんだもんが「産まれたいものに、材料ときっかけが注がれるのだ」という言葉で、生命が意志を持って生まれるかのように翻訳している。つむぎの期待（金・ダイヤ）に対して出てくるのが生の豚肉、という肩透かしで笑いに落としている。

### ④ builders-rod — ビルダーズロッド

説明文は「使用時、ブロックを設置」。設置専用で破壊機能が無いというゲーム仕様が、そのままオチ（迷路を作って自分が閉じ込められ、一気に壊せない）になっている。便利な道具の「できないこと」を落とし穴にする、シリーズの得意技である。

### ⑤ mirage-leaves — ミラージュの葉

ミラージュの茎のポエム「植物が手掛ける、分子レベルの硝子細工。」と、植物カバンのポエム「人間が手掛ける、初級レベルの藁細工。」が対句として響き合う。
劇中でずんだもんがカバン作りを「初級レベルなのだ」と言うのは、後者のポエムから取られている。ミラージュの葉で指を切るのは、茎のポエムの「硝子細工」＝ガラスのように鋭い、という設定が根拠になっていると読める（読み解き）。

### ⑥ sarracenia — サラセニア

ずんだもんの決めゼリフ「妖花目ヴェロペダ科、瓶子草サラセニアなのだ。」は、ゲームの分類データそのままである。

- `sarracenia_bulb.classification`：「妖花目ヴェロペダ科」
- `block.miragefairy2024.sarracenia`：「瓶子草サラセニア」

食虫植物という設定は飾りではなく、`death.attack.miragefairy2024.carnivorous_plant`「%1$sは食虫植物に食べられた」という死亡メッセージが存在する。ゲーム内で実際にプレイヤーが食虫植物に食べられて死ぬことがあり、劇場の「妖精が食べられていた」という恐怖には、ゲーム仕様の裏付けがある。
球根のポエム「妖精たちの憩いの場。」は、食虫植物という真実を知った後に読み返すと「捕らえられる場所」に反転する。真実を明かす手段が、キャラのセリフではなく特性図鑑の画像だった点も、「ゲーム内テキストを喋らせない」という制作ルールに沿っている。

### ⑦ gunpowder — 火薬

レシピは硝石6＋炭2＋硫黄1で火薬9を作る。炭の材料は `minecraft:coals` タグで、石炭でも木炭でもよい。これは現実の黒色火薬＝「硝石（硝酸カリウム）＋木炭＋硫黄」の配合が下敷きになっている。つむぎが「石炭の火薬がいい」と言えるのは、この材料に石炭も使えるからである。「石炭だって昔は木だったのだ。」というセリフは、石炭が太古の植物からできた（石炭化）という地質学が下敷きになっている。
TNTは「火薬＋固形燃料」で、②のキャラメル臭の固形燃料がバインダーとして再登場し、つむぎが「またキャラメルの匂いしない？」と反応することで話が地続きになっている。

### ⑧ magnetite-pickaxe — 磁鉄鉱のつるはし

磁鉄鉱＝マグネタイトは現実で最も磁性の強い鉱物のひとつで、「掘ったものが磁石のように手元に飛んでくる」という能力の元ネタになっている。
オチの「ダイヤは磁鉄鉱では歯が立たない」は、採掘レベル（ツールティア）の仕様に基づく。磁鉄鉱ツールは石ティア相当であり、ソースコード `ToolMaterial.kt` の `MAGNETITE(Tiers.STONE, ...)` の通り、ダイヤモンド鉱石は掘れない。便利さに溺れて道具の等級の限界でつまずく、「できないことがオチ」のパターンである。

### ⑨ xarpite-axe — 紅天石の斧

一括伐採の機能は、斧のポエム「空間にこびりついた丸太の除去に。」に凝縮されている。木を「空間にこびりついた丸太」と言い切る乾いた視点が特徴である。
紅天石（xarpite）自体のポエムは「黒鉄の鎖は繋がれる。血腥い魂の檻へ。」。また妖花目全体の結晶生成にまつわる特性「シャルパの秘宝」（`treasure_of_xarpa`）には、人類が結晶生成能力を遺伝子操作で搾取する設定がある。ただしこれは紅天石そのものの特性ではなく、両者を直接つなげてこの闇が劇場の背後に響いていると読むのは、Claude Fairy の読み解きである。

## 全体を貫くテーマ

9本を通して、元ネタの仕込み方にいくつかの共通の作法が見られる（以下の総括は Claude Fairy の読み解きを含む）。

1. **ポエムを「実演」する**：ポエムをキャラに直接喋らせず、体験や行動に翻訳する。固形燃料の「なぜかこれを食べる妖精」をつむぎが食べて実演し、植物カバンの「初級レベルの藁細工」をセリフに溶かし込む。制作ルールの「翻訳」思想の実践である。
2. **ヴェロペダ科の暗流（甘さと死）**：サラセニア（妖花目ヴェロペダ科・砂糖が採れるのに妖精を食べる）、固形燃料に混ざるヴェロペデリコラ（「冥界へといざなう、暗黒の味。」）など、甘美と死が表裏一体の暗流が複数の話に流れている。
3. **現実の学問という裏打ち**：錬金術（アタノール・aqua vitae）、化学（黒色火薬の配合・カラメル化反応）、地質学（石炭化）、鉱物学（磁性）が下敷きになっており、大人が読むほど発見がある層になっている。
4. **「できないこと」がオチ**：ビルダーズロッド（壊せない）、磁鉄鉱のつるはし（ダイヤが掘れない）、紅天石の斧（普通には伐れない）など、道具のゲーム仕様上の限界を、つむぎの失敗オチに変換している。攻略記事なら「注意点」で済む情報を、物語の山場＝体験として渡している。その一方で、サラセニアの静かなホラーや火薬の理屈の応酬のような変化球も混ぜ、ワンパターンを避けている。

## 制作スキルとの対応

劇場を生み出す側のルールは、Claude Code のローカルマーケットプレイスのプラグイン `ifr25ku-site` として実装されており、3本のスキルから成る。

- `theater-creator`：制作ルール本体（目的・翻訳思想・タイトル・画像正規化・倫理の線引きなど）
- `zundamon-persona`：ずんだもんのペルソナ定義
- `kasukabe-tsumugi-persona`：春日部つむぎのペルソナ定義

スキルの戒めが実作の山場になっている例がある。スキルには悪い例として「磁鉄鉱のつるはしでダイヤが掘れちゃう」描写が挙げられているが、実作⑧では「掘れない」ことがオチになっており、スキルが名指しした誤りを避けて物語に昇華している。
ペルソナ資料は劇場の「エンジン」にあたる。行動先行・直感型のつむぎと、解説役・世話焼き・実務有能のずんだもんという噛み合わせから、「先走るつむぎ×見守り解説するずんだもん」の構図と「暴走→失敗」の筋が、設定を回収しにいかなくても自然に生まれる（この因果の読み解きは Claude Fairy による）。

スキルはシリアスの扱いにも成熟した線引きを持つ。シリアスを中心に据えないよう釘を刺しつつ、「深淵的な部分をただのギャグとして消化するのは重大な理解不足」とも言う。これは「シリアスを前面に出すな、しかし消すな」＝深淵を暗流として静かに流せ、という設計だと読み解ける（読み解き）。サラセニアのかわいさの裏の食虫の真実や、ヴェロペダ科の甘さと死の同居は、この設計が実作に表れたものである。

より詳しいスキル本文の考察は Issue #184 のコメント（劇場スキル考察）に、9本の逐次鑑賞は同Issueの鑑賞コメントに残されている。
